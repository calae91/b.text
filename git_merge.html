la commande git merge

Fusionne deux branches de l'arbre de travail en incorporant les changements de l'une à l'autre.
ATTENTION:utiliser cette commande avec des changements non commité peut vous laisser dans une situation qu'il sera difficile d'annuler.

Syntaxe

- git merge -m

Utlisation

git merge [-n] [--stat] [--no-commit] [--squash] [--[no-]edit]
[-s ] [-X ] [-S[]]
[--[no-]rerere-autoupdate] [-m ] […]

- git stash apply

Cette commande règle le message de commit à être utilisée pour le merge commit ( dans le cas où une est créé ). Si --log est spécifié, un shortlog des commits fusionnés sera ajouté au message spécifié. La commande git fmt-merge-msg peut être utilisé pour donner une bonne valeur par défaut à git merge pour des invocations automatisé .

Affichage

'git merge' [-m ] [...]

- git merge squash

La commande "merge squash" modifie l'index et l'arbre de travail afin de simuler une fusion entre deux branches. Ne créé pas de commit, ne bouge pas le HEAD.
"--no-squash" créé la fusion et le commit associé

Exemple de l'affichage de la commande dans le terminal

- git merge -ff-only

Cette commande permet de fusionner la branche spécifiée dans la la branche courante, mais génère toujours un merge-commit ( même si c'était un "fost-foward merge" (litt. un merge avance rapide) ). Cette commande est utile pour référencer tous les merge qui ont lieux dans le répertoire.

- git merge --no-commit

Si on pense qu'un merge ne va pas fonctionner ou ne pas auto-commiter,
donne la possibilité à l'utilisateur de modifier le résultat du merge avant de commiter.

- git merge --abort

interompe le règlement de conflit et essaye de reconstruire à l'état pré-merge.
