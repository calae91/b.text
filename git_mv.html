la commande git mv

git mv permet de renommer des fichiers, un répertoire ou un lien symbolique
(sauvegarde/supprime/créer). On peut aussi renommer sans avoir à utiliser la commande mv.

Syntaxe

- git mv "toto.txt tata.txt"

Affichage

# On branch master
# Changes to be committed:
# (use "git reset HEAD ..." to unstage)
#
# renamed: toto.txt -> tata.txt
#

- git mv -f

ATENTION cette commande force le changement de nom d'un dossier et detruit les dossiers existant sous le même nom.

Affichage

user@lubuntu-13:~/Bureau$ ls
Documents titi.txt toto.txt zfejejksdhficoqklsc
user@lubuntu-13:~/Bureau$ git mv -f toto.txt titi.txt
user@lubuntu-13:~/Bureau$ ls
Documents titi.txt zfejejksdhficoqklsc

- git mv -n "fichier ancien" "fichier nouveau"
la commande permet de montrer les résultats sans faire aucun changement. .

Affichage

Checking rename of 't.txt' to 'a.txt'
Renaming t.txt to a.txt
user@lubuntu-13:~/Bureau$ ls
Documents t.txt
